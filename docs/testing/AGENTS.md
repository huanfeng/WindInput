### test-word-frequency.md

词频学习功能的专项测试。

**测试覆盖**：
- 学习机制：输入同一词语多次后排序提升
- 遗忘规则：较长时间未使用的词语权重衰减
- 导出验证：导出的学习数据格式正确

**测试步骤**：
1. 重置词库
2. 输入指定词语序列（N 次）
3. 检查排序变化
4. 等待遗忘时间过期
5. 验证权重衰减
6. 导出并检查数据格式

**验收标准**：
- [ ] 学习效果明显（相同词语输入 3 次后排序靠前）
- [ ] 遗忘机制正常（7 天未使用后权重衰减）
- [ ] 导出格式正确（JSON/YAML）

**检查项**：

| 快捷键冲突检测 | 设置 Ctrl+A 为快捷键 | 提示"冲突：系统快捷键" | ☐ 通过 |

## 工作指南

### 执行测试

**完整测试流程**：
```bash
# 1. 阅读测试指南
docs/testing/testing-guide.md

# 2. 按章节逐个测试
# (参考 testing-guide.md 的每一章)

# 3. 记录结果
# (使用 test-helper.html 或电子表格)

# 4. 词频功能专项测试
docs/testing/test-word-frequency.md

# 5. 配置界面验收
docs/testing/settings-checklist.md

# 6. 汇总报告
# (通过/失败/已知问题)
```

### 针对新功能的测试

当添加新功能时：
1. 在 `testing-guide.md` 的适当章节添加测试场景
2. 定义预期输出和边界情况
3. 在代码审查前完成测试
4. 如发现缺陷，关联 GitHub Issue

**示例**：
```markdown
### 新增：自动标点配对

测试场景：
- 输入左括号 ( 时自动补全右括号 )
- 确认中括号、花括号、双引号也支持

操作：
1. 在文本框中输入 hello（
2. 期望：自动补全为 hello()，光标在括号内

边界情况：
- 已有右括号时不重复补全
- 中英文混输时正确识别
```

### 发现缺陷时

1. **复现缺陷**：记录确切的操作步骤
2. **截图/视频**：视觉证据有助于定位
3. **系统信息**：Windows 版本、输入法方案、DPI 等
4. **创建 Issue**：标题简洁，描述完整

**示例**：
```markdown
## Bug: 五笔拼音混输时某些词汇排序错误

**复现步骤**：
1. 切换到"五笔拼音混输"方案
2. 输入"nihao"（拼音代码）
3. 按 Tab 切换为五笔候选

**预期**：显示五笔码表的候选词语

**实际**：候选词语排序错误，常用词在后

**系统**：Windows 11, 全拼方案, 96 DPI

**附件**：[screenshot.png](...)
```
