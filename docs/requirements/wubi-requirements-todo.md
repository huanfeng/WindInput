## P0 必须支持（15项）

### 已实现 ✅

- [x] **R02 自定义标点** — 已支持 `input.punct_custom` 自定义映射，后端转换逻辑和设置界面均已落地
- [x] **R04 数字后句号行为可配置** — 已支持 `input.smart_punct_after_digit`
- [x] **R05 四码唯一自动上屏可配置** — `auto_commit_unique`
- [x] **R06 第五码顶字上屏可配置** — `top_code_commit`
- [x] **R07 纯五笔模式** — `wubi86` 方案为纯五笔
- [x] **R08 候选数量可配置** — `ui.candidates_per_page`，支持 `0` 选第 10 项
- [x] **R09 分号/引号选重** — `select_key_groups = ["semicolon_quote"]`
- [x] **R10 中英文切换键** — 支持左右 `Shift`、左右 `Ctrl`、`CapsLock`
- [x] **R11 切换到英文时编码处理** — `hotkeys.commit_on_switch`
- [x] **R14 z 键重复录入** — `z_key_repeat`

### 部分实现 🔶

- [ ] **R01 词库导入/导出** — 用户词库、用户短语、系统短语均支持导入导出；但”系统词库应支持多种类型词条并统一迁移维护”的目标尚未完全达到
- [ ] **R03 Shift+字母行为可配置** — 当前只有 `shift_temp_english.enabled` 开关；缺少”临时英文”和”大写字母直接上屏”两种模式的明确切换配置
- [ ] **R12 临时拼音/临时生僻字引导** — 临时拼音入口已实现且触发键可配置；但生僻字引导由临时拼音间接覆盖，无独立入口
- [ ] **R13 回车键行为** — 当前固定为”确认段 + 原始编码上屏”；尚不支持在”编码上屏 / 清空编码”之间切换

### 未实现 ❌

- [ ] **R15 单候选时选重键行为** — 单候选时 `;` / `'` 不会执行 2/3 选，但会落入标点处理；在默认 `punct_commit` 开启时，仍可能形成“首选 + 标点”上屏，不符合“最好不起作用”的要求

## 当前优先级建议

### 第一优先：仍有明显差距的 P0

1. **R01 词库导入/导出完善**
   当前导入导出能力分散在用户词库/短语层，仍缺少面向“完整五笔使用数据迁移”的统一方案

   增加“临时英文 / 直接大写上屏”显式选项，避免只有布尔开关

   增加“编码上屏 / 清空编码”选项
4. **R15 单候选时选重键无动作**
   需要避免默认 `;` / `'` 误触形成“首选 + 标点”上屏

   增加用户可维护的黑白名单或标记层

1. **R26 词库维护平台化**
2. **R27 生僻字词标记快捷键**
3. **R28 更细粒度快捷键自定义**
