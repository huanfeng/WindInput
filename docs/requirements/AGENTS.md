# requirements/ - 需求规格与功能规划

## 用途

功能需求、规格定义和实现进度跟踪。这些文档定义功能范围、优先级和验收标准，指导开发工作。

## 关键文件

| 文件 | 描述 |
|------|------|
| `features-todo.md` | 功能规划总表：所有计划功能的列表、优先级、实现状态、所属模块 |
| `wubi-requirements.md` | 五笔功能需求分级：P0（必须）、P1（建议）、P2（可选）、P3（暂不需要） |
| `wubi-requirements-todo.md` | 五笔需求实现进度跟踪：完成度、测试状态、已知问题 |

## 文件说明

### features-todo.md

功能规划的总表，包含所有计划中的功能。

**结构**：

| 功能 | 模块 | 优先级 | 状态 | 测试 | 备注 |
|------|------|--------|------|------|------|
| 快速输入（词库预定义） | Pinyin | P1 | 规划中 | 待设计 | design/ 中的方案 |
| 自动标点配对 | Common | P2 | 设计完成 | 待开发 | 括号、引号等 |
| 五笔拼音混输 | Wubi | P1 | 开发中 | - | schema 已完成 |

**优先级定义**：
- **P0**：核心功能，项目交付必需
- **P1**：主要功能，计划版本中优先实现
- **P2**：增强功能，视时间和资源情况实现
- **P3**：未来方向，暂不规划

**状态定义**：
- `规划中` - 需求未确定
- `设计中` - 正在进行设计评审
- `设计完成` - 设计文档已完成，待开发
- `开发中` - 代码实现中
- `待测试` - 开发完成，待功能测试
- `已完成` - 测试通过，已发布

### wubi-requirements.md

五笔输入法的需求分级。

**P0 - 必须支持**：
- 标准五笔86 输入
- 自动上屏（码长自动上屏）
- 多字词库支持
- 基础词频学习

**P1 - 建议支持**：
- 拼音混输（五笔拼音互补）
- 符号直通车（简码输入符号）
- 自定义快捷键
- 词库导入（用户自定义词库）

**P2 - 可选**：
- 五笔笔画输入（未知字用笔画代替）
- 动态码表调整（根据使用习惯）

**P3 - 暂不需要**：
- 其他输入法方案（郑码、仓颉等）

## 工作指南

### 添加新需求

1. **明确功能范围**：
   - 用户故事（User Story）：描述使用场景
   - 验收标准（Acceptance Criteria）：功能如何验证

2. **分配优先级**：参考 P0-P3 定义和当前产品阶段

3. **更新文件**：
   - 主功能表 → `features-todo.md`
   - 五笔相关 → `wubi-requirements-todo.md`

4. **示例需求**：
   ```markdown
   ### 自定义快捷键（P1）
   
   **用户故事**：作为用户，我希望自定义快捷键（如 Ctrl+Q 快速切换输入法），以提升工作效率。
   
   **验收标准**：
   - [ ] 配置界面支持快捷键编辑
   - [ ] 支持的快捷键列表清晰（避免与系统/应用冲突）
   - [ ] 重启后快捷键仍生效
   - [ ] 重复定义时提示冲突
   
   **涉及模块**：wind_setting（UI）、wind_input（RPC 服务）
   ```

### 跟踪实现进度

1. **更新状态**：当功能实现进展时更新"状态"列
2. **关联 Issue**：在"Issue"列填入 GitHub Issue 编号
3. **记录阻塞**：如遇阻碍，在备注列说明

### 发布版本时

1. 标记该版本包含的功能为"已完成"
2. 生成发布说明（docs/release-notes/vX.Y.Z.md）
3. 检查所有 P0 功能已测试通过
